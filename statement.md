# Problem Statement

The Traditional method or operations of Banking face several challenges. For example the manual account management leads to errors due to human interventioins. 
Thereafter these is issue of inefficient transaction processing and tracking, then there are poor customer experience due to long waiting hours, lack of real time account information fetching and viewing, also they face difficulties in maintaining transaction history. So the students as well as the campus staff need a simple, reliable system to manage bank accounts and transactions. Due to a lot of students in university it is needed to be highly scalable, secure, maintainable and extensible. Our solution really looks forward solving these issues using java and OOPS core features and their implementations.


# Scope of the Project

Scope of the project is to implement the core account and transaction features for example creating account, depositing, withdrawing, transfering value. user should also be able to track their transaction hsitory with timestamps and should check their Account balance after transaction. The system also helps them by providing the summary views for example all accounts, all students, bank totals etc. This keeps the implementation lightweight and console-based (no database or web UI in this version).

# Target Users

- VIT Bhopal students who need to manage personal accounts.
- Campus staff who help maintain student accounts.
- Instructors and students using the project for learning and demonstration.

# High-level Features

- Student account creation and management.
- Transaction history with timestamps and transaction IDs.
- Multiple account types: Savings, Current, Salary.
- Deposit, withdrawal, and transfer operations with validation.
- Bank summary: total students, total accounts, and total deposits.
- Simple console-based menu for interactive use and testing.
